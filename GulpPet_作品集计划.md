# GulpPet · 咕嘟 — 产品作品集计划
> 投递方向：产品经理实习 | 目标：字节跳动 / 腾讯 / 阿里等大厂
> 起始日期：2026-06-18 | 计划周期：2周

---

## 一、项目概述

| 项目 | 内容 |
|---|---|
| 产品名称 | GulpPet · 咕嘟 |
| 一句话定位 | 懂你工作节奏的补水小伙伴 |
| 核心用户 | 长期在电脑前工作的知识工作者 |
| 核心问题 | 补水靠意志力、提醒太打扰、记录太麻烦 |
| 解决方案 | 桌面灵宠 + 工作状态感知 + 低打扰智能提醒 |

## 二、作品集展示目标

大厂 PM 面试最关注以下五个维度，本次作品集逐一对应：

| 维度 | 对应交付物 |
|---|---|
| 用户洞察力 | 竞品分析报告 + 用户研究报告 + Persona |

## 三、2 周行动计划

### 第一周：研究与定义（Day 1–7）

#### Day 1–2：精简用户研究（Lean Research）

**策略说明：** 本项目已完成扎实的二手研究（文献综述 + 科学依据），无需进行大规模用户调研。精简研究聚焦三件事：竞品分析 + 3 人访谈 + 2 个 Persona，合计约 2 天，为 PRD 提供一手洞察。

**✅ 已完成（二手研究 / 理论基础）：**
- [x] 中国居民膳食指南 2022 每日饮水量推荐标准
- [x] IOM 性别差异饮水推荐
- [x] Kenney & Chiu 2001（65岁以上口渴感减弱）
- [x] 提醒频次算法设计（时间驱动 + 少量多次原则）
- [x] 杯型系统设计依据
- [x] 初始化表单字段与决策依据

**待完成（一手研究）：**
- [ ] 竞品分析（WaterMinder、Plant Nanny、Hidrate Spark、Forest、潮汐）
  - 功能对比、核心差异、留存痛点
  - 重点：为什么用户用着用着就放弃了
- [ ] 设计访谈提纲（5–6 个问题，聚焦行为而非态度）
- [ ] 完成 3 人访谈（身边长期用电脑的朋友/同学，每人 20 分钟）
- [ ] 基于访谈 + 已有理论框架，产出 2 个 Persona

**输出文件：**
- `01_用户研究/产品科学依据与背景研究.md` ✅（已完成）
- `01_用户研究/竞品分析报告.docx`
- `01_用户研究/用户研究报告.docx`（含 Persona + 访谈摘录）

#### Day 3–4：产品需求文档（PRD）
**目标：** 完整定义第一版产品，展示结构化产品思维。

- [ ] 产品背景与目标（为什么做、做给谁、成功定义）
- [ ] 用户故事（User Stories，覆盖核心使用场景）
- [ ] 功能需求清单（MoSCoW 优先级：Must / Should / Could / Won't）
  - Must：桌宠浮窗、智能提醒、一键记录、免打扰规则
  - Should：宠物状态变化、提醒语气选择
  - Could：周报、多宠物、主题皮肤
  - Won't（V1不做）：智能水杯接入、手机端
- [ ] 信息架构与核心用户流程
- [ ] 数据指标体系
  - 核心指标：DAU提醒响应率、7日留存率
  - 健康指标：平均每日提醒次数、用户设置完成率
  - 护栏指标：提醒被关闭比例（不能过高）
- [ ] 非功能需求（性能、隐私、权限）

**输出文件：**
- `02_产品文档/GulpPet_PRD_v1.0.docx`

## 五、大厂 PM 面试应答框架

拿到这个项目在面试中被问到时，建议用以下结构回答：

```
① Why this problem?
   → 用用户研究数据说明：现有喝水提醒产品的核心问题不是"提醒"，而是"打扰"

② Why this solution?
   → 竞品对比 + 取舍逻辑：为什么选桌宠形式，而不是手机通知/健康手环

③ How did you prioritize?
   → MoSCoW 框架，说明 V1 砍掉了哪些功能，判断依据是什么

④ How do you measure success?
   → 7日留存 + 提醒响应率 + 提醒关闭率（护栏）

⑤ What would you do next?
   → 根据 MVP 数据，V2 加入智能手机端 / V3 接入硬件
```

## 六、风险与注意事项

| 风险 | 应对 |
|---|---|
| MVP 开发时间不够 | 优先完成文档和原型，MVP 功能可以只做最核心的 3 个 |
| 访谈对象难找 | 3人即可，重质量不重数量，身边同学/朋友均可 |
| Figma 不熟练 | 低保真用任意工具（甚至手绘扫描），高保真优先做最核心的2-3个页面 |
| 文档太厚没人看 | 所有文档控制在"10分钟可以通读"的篇幅，能图表不写段落 |
