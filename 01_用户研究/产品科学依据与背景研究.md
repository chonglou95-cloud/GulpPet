## 一、每日饮水量推荐标准

### 1.1 中国居民膳食指南 2022（中国膳食指南委员会）

**参考来源：** 中国营养学会.《中国居民膳食指南（2022）》. 人民卫生出版社, 2022.

中国 DG 2022 针对"纯饮用水"（不含食物中水分）的推荐量：

| 人群 | 推荐饮水量（纯饮用水） |
|---|---|
| 成年男性（轻体力活动） | 1700 ml/天 |
| 成年女性（轻体力活动） | 1500 ml/天 |
| 65岁以上老年人 | 1500 ml/天（需主动饮水，参见第三节） |

> 注：以上数字仅指"纯饮用水"，不含膳食中的水分摄入（约 700–1000 ml）。

**活动强度分级对应调整：**

| 活动强度 | 定义 | 调整幅度 |
|---|---|---|
| 轻度（久坐） | 办公、学习，每天步行 < 5000 步 | 基准值（1500–1700 ml） |
| 中度 | 快走、骑行、日常轻微体力劳动 | +300–500 ml |
| 重度 | 运动员训练、重体力劳动、高强度健身 | +500–1000 ml 或更多 |

**产品应用：** 初始化表单收集用户活动强度，以此确定推荐目标区间的基准值（轻度 = 中国 DG 2022 推荐值，中度 / 重度依次上调）。

---

### 1.2 美国医学研究院（IOM）推荐值

**参考来源：** Institute of Medicine. *Dietary Reference Intakes for Water, Potassium, Sodium, Chloride, and Sulfate.* National Academies Press, 2005.

| 人群 | 总水摄入（包含食物） | 估算纯饮水（80%来源于饮品） |
|---|---|---|
| 成年男性（19–50岁） | 3.7 L/天 | ≈ 2.9–3.0 L |
| 成年女性（19–50岁） | 2.7 L/天 | ≈ 2.1–2.2 L |

> **说明：** IOM 的 3.7 L / 2.7 L 是"总水摄入"，约 20% 来自固体食物，因此纯饮水量约为总量的 80%。  
> 中国 DG 2022 与 IOM 的差异在于：DG 2022 仅统计纯饮用水，两套体系在换算后基本一致。

---

### 1.3 "体重 × 30ml" 公式的说明

临床营养实践中常用的经验公式"每日饮水量 = 体重（kg）× 30 ml"是一种**临床便捷估算法**，源自教科书级别的经验总结，而非专项科学研究的产物。其优点是计算简便，缺点是忽略了个体活动强度、年龄、气候等影响因素。

**GulpPet 的选择：** 以中国 DG 2022 活动强度区间为主要计算基础，不在产品界面展示"体重 × 30 ml"，避免误导用户。仍在初始化表单收集体重，用于未来版本的更精细化计算。

## 二、性别对水需求的影响

**参考来源：** IOM 2005（同上）；以及体成分差异的相关研究。

男性与女性每日推荐饮水量相差约 25–30%，主要原因包括：

- 男性平均体重和肌肉量更高（肌肉组织含水量 70–75%，脂肪组织含水量约 10%）
- 男性基础代谢率更高，产热更多，散热需水量更大

**产品应用：** 初始化表单必须收集性别，用于区分推荐目标区间：
- 男性：1700 ml（轻度）
- 女性：1500 ml（轻度）

## 三、年龄分段的理论依据

### 3.1 中国 DG 2022 年龄分段框架

中国 DG 2022 对成年人的年龄分段为：
- 18–49 岁（青中年成年人）
- 50–64 岁（中老年人）
- 65 岁及以上（老年人）

GulpPet 的年龄段选项直接采用此分段，确保具有可引用的官方依据。

### 3.2 65 岁以上老年人的特殊处理

**参考来源：** Kenney, W.L. & Chiu, P. (2001). Influence of age on thirst and fluid intake. *Medicine & Science in Sports & Exercise*, 33(9), 1524–1532. PMID: 11528344.

该研究发现：老年人（≥65 岁）的口渴感知机制随年龄退化，存在**生理性口渴感减弱**现象——即身体已经出现轻度脱水，但主观上不感到口渴。

**产品应用（65岁以上特殊模式）：**

| 常规模式 | 65岁+ 特殊模式 |
|---|---|
| 基于活动强度区间推荐目标 | 推荐目标 ≥ 1500 ml（DG 2022 老年人标准） |
| 基础提醒间隔：工作窗口 ÷ 8 | 提醒间隔缩短约 15–20%（更频繁提醒） |
| 提醒语气：轻松友好 | 提醒语气：关爱提示，强调主动补水的重要性 |

## 四、核心提醒算法设计

### 4.1 提醒频次的科学依据

**理论支撑：**

中国 DG 2022 明确建议"少量多次"的饮水方式，而非集中大量饮水。具体建议：
- 早晨起床 1 杯（约 200 ml）
- 睡前不宜大量饮水
- 其余水分均匀分布在清醒时段

"均匀分布"映射到产品逻辑：若有效工作时段为 8 小时，理论上每小时提醒 1 次，共 8 次提醒。

**约束条件：**

| 参数 | 范围 | 理由 |
|---|---|---|
| 最短间隔 | 30 分钟 | 低于此值会显著打扰工作流 |
| 最长间隔 | 90 分钟 | 超过此值有失去提醒效果的风险 |
| 基础间隔 | 工作窗口时长（分钟）÷ 8 | 对应"少量多次、均匀分布"原则 |

**基础间隔计算示例：**

```
用户工作时段：09:00 – 18:00（540 分钟）
基础间隔 = 540 ÷ 8 = 67.5 分钟 → 约 68 分钟
是否在 [30, 90] 范围内？是 → 直接使用
```

```
用户工作时段：20:00 – 24:00（240 分钟）
基础间隔 = 240 ÷ 8 = 30 分钟 → 正好是下限，使用 30 分钟
```

### 4.2 重要原则：提醒间隔与杯型/饮水量完全解耦

杯型（杯子容量）和实际记录的饮水量**不影响**提醒间隔的计算。

原因：用户每次喝水量高度不确定——有时大口喝完一整杯，有时只是抿几口。如果把提醒频次绑定到"每次喝多少毫升"，逻辑会无法收束。

**正确的逻辑分工：**
- 提醒间隔 → 完全由时间驱动
- 饮水记录 → 追踪总摄入量（仅供统计和展示）
- 杯型 → 仅用于快速记录的UI便捷操作（"我喝了一杯350ml"）

## 五、用户记录饮水后的重置逻辑

当用户收到提醒并主动记录饮水（点击"已喝水"）后，下一次提醒的时间如何计算？

### 5.1 重置逻辑规则

```
响应时间差 = 用户记录时刻 - 上次提醒时刻

if 响应时间差 < 基础间隔 × 0.5:
    # 用户很快响应，说明刚被提醒就喝了
    下次提醒 = 记录时刻 + 基础间隔

elif 响应时间差 < 基础间隔:
    # 用户在合理时间内响应
    下次提醒 = 记录时刻 + 基础间隔

else:
    # 用户延迟响应（可能是"稍后"后才喝）
    # 不惩罚，仍从记录时刻重新计算
    下次提醒 = 记录时刻 + 基础间隔
```

**关键决策：始终从"记录时刻"重新计时，而不是从"提醒时刻"。**

原因：从用户的身体感受来说，补水是以实际喝水的时刻为准，不是以提醒的时刻为准。

### 5.2 "稍后"操作的处理

| 用户操作 | 逻辑 |
|---|---|
| 已喝水 | 重置计时器，从记录时刻 + 基础间隔后再次提醒 |
| 稍后15分钟 | 15分钟后再次提醒，不重置今日喝水记录 |
| 跳过 | 计时器照常运行，不记录饮水，不影响后续提醒 |

### 5.3 "免打扰模式"的处理

| 触发条件 | 行为 |
|---|---|
| 用户进入全屏模式（演示/视频） | 提醒静默，计时器暂停 |
| 用户标记的免打扰时段 | 跳过该时段的提醒，退出后恢复计时 |
| 桌面端关闭 | 计时器暂停，手机端接管提醒（V1阶段由用户手动在手机端开启） |

## 六、初始化表单字段设计

### 6.1 字段清单与决策依据

| 字段 | 类型 | 必填 | 科学依据 |
|---|---|---|---|
| 性别 | 单选（男/女/不透露） | 是 | IOM 2005：男女推荐量差异约 25% |
| 年龄段 | 单选 | 是 | 中国 DG 2022 年龄分段；Kenney & Chiu 2001（65+特殊模式） |
| 体重 | 数字输入 | 否（可跳过） | 用于未来精细化计算；V1 用活动强度替代 |
| 活动强度 | 单选（轻度/中度/重度） | 是 | 中国 DG 2022 活动强度分级 |
| 主要使用电脑的时段 | 时间范围选择（开始-结束） | 仅桌面端 | 用于计算提醒间隔 |
| 使用设备 | 单选（仅桌面/桌面+手机） | 是 | 决定是否收集"电脑时段"字段 |

### 6.2 年龄段选项

```
□ 18–49岁（青中年）     ← 标准推荐值
□ 50–64岁（中老年）     ← 标准推荐值，未来版本可微调
□ 65岁及以上            ← 启用特殊模式（更频繁提醒 + 关爱语气）
```

> **设计原则：** 不询问具体年龄，只询问年龄段，降低用户隐私顾虑。年龄段划分直接对应中国 DG 2022 官方分段，不随意自定义。

### 6.3 活动强度引导文案

避免使用专业术语"轻度/中度/重度"，用具体场景描述：

```
📋 大部分时间坐着（办公室工作、学习、开会）
🚶 偶尔起身走动（每天有一定步行、做家务）  
🏋️ 经常运动或体力活动（健身、户外、体力劳动）
```

### 6.4 "主要使用电脑时段"的场景分叉

```
if 设备选择 == "仅桌面端" or "桌面+手机":
    显示: 请选择你主要用电脑工作或学习的时段
    字段: 开始时间（时:分） + 结束时间（时:分）
    说明文案: "我们会在这段时间内为你安排提醒，不在时段内不打扰"

if 设备选择 == "仅手机端":
    不显示该字段（没有"用电脑时段"概念）
    改为: 提醒活跃时段（早上 X 点到晚上 X 点）
```

> **产品策略：** 在设备选择页面，视觉上突出"桌面端"和"桌面+手机"选项（更大的卡片），并注明"🐾 桌宠功能仅在桌面端可用"，有意识引导用户不选择纯手机端。

## 七、杯型系统设计依据

### 7.1 设计目标

允许用户一键记录喝水，无需每次手动输入毫升数，同时解决"用户不可能每次都喝固定容量"的问题。

### 7.2 预设杯型参考

| 杯型 | 容量参考 | 场景描述 |
|---|---|---|
| 一次性杯（小） | 200 ml | 纸杯、塑料杯，通常 1–2 口 |
| 玻璃杯·马克杯（中） | 350 ml | 玻璃杯或陶瓷马克杯，类似宜家 POKAL |
| 保温杯（大） | 500 ml | 常见随行杯 / 保温瓶 |
| 矿泉水瓶 | 550 ml | 农夫山泉 550ml / 怡宝 555ml |
| 自定义杯型 | 用户输入 | 支持命名 + 毫升数 |

> 最多保存 5 个杯型；记录界面默认展示上次使用的杯型在最顶部。

### 7.3 记录交互的核心设计

- 提醒弹窗弹出时，直接展示杯型快选按钮（一键记录）
- 记录量是"大概量"，不追求精确，减轻用户负担
- 后台记录 `HydrationEvent`，但展示给用户的是累计百分比（今日目标完成度）

## 八、技术架构说明（与算法相关部分）

### 8.1 数据结构

```typescript
type HydrationEvent = {
  kind: "suggestion"    // 系统发出提醒
       | "response"     // 用户响应（已喝水/稍后/跳过）
       | "measurement"  // 用户主动记录（不经过提醒）
  amountMl?: number     // 仅 response/measurement 有值
  source: "desktop" | "mobile" | "manual"
  confidence?: number   // 未来：智能水杯数据的置信度
  timestamp: string     // ISO 8601
}
```

### 8.2 目标计算逻辑（伪代码）

```python
def calculate_daily_goal(gender, age_group, activity_level):
    # 基础目标（中国 DG 2022）
    if gender == "male":
        base = 1700  # ml
    else:
        base = 1500  # ml

    # 活动强度调整
    if activity_level == "moderate":
        base += 400
    elif activity_level == "heavy":
        base += 700

    # 年龄段调整
    if age_group == "65+":
        # Kenney & Chiu 2001：强调老年人即使不渴也需要保证摄入量
        base = max(base, 1500)

    return base

def calculate_base_interval(work_start, work_end):
    window_minutes = (work_end - work_start).total_minutes()
    interval = window_minutes / 8

    # 约束在 [30, 90] 分钟之间
    return max(30, min(90, interval))
```
